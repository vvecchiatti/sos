# Söuthsea Swimrun — website

The website for Söuthsea Swimrun (the UK's first swimrun club), live at
**https://southseaswimrun.com**.

It's a static site — plain HTML/CSS with one small JS file. No build step,
no framework, no database.

## Where everything lives / how it goes live

| Thing | Where |
|---|---|
| **Source code** | This GitHub repo (`southseaswimrun/sos`), branch `main`, in `southsea-site/` |
| **Hosting** | Cloudflare Pages — auto-deploys from this repo |
| **Domain + DNS** | Cloudflare (`dash.cloudflare.com`) |
| **Membership / sessions / payments** | Heylo (`heylo.com`) |

**To change the site:** edit a file in `southsea-site/`, commit, and push to
`main`. Cloudflare rebuilds automatically and the change is live in ~1 minute.

## Structure

```
southsea-site/
├── index.html          # Homepage (hero, sessions, partners, team, latest posts)
├── blog.html           # Race reports & blog index — GENERATED, do not hand-edit
├── admin.html          # Blog admin (/admin) — write & publish posts
├── posts/
│   ├── posts.json      # Post manifest — the source of truth for listings
│   └── *.html          # One HTML file per post (new ones generated by /admin)
├── images/
│   ├── partners/       # Partner logos (partners grid on the homepage)
│   ├── team/           # Team avatar photos (vini/sarah/adrian/chris .jpg)
│   └── posts/<slug>/   # Photos uploaded through /admin, one folder per post
├── js/main.js          # Nav hamburger, fade-in, gallery lightbox, year
├── js/templates.js     # Shared HTML generators (blog page, posts, sitemap)
├── js/admin.js         # Admin logic (GitHub API publishing)
├── favicon.svg         # Site icon
├── 404.html            # Branded "not found" page
├── robots.txt          # Search-engine directives
├── sitemap.xml         # GENERATED from posts.json
├── _headers            # Security + caching headers (Cloudflare Pages)
└── _redirects          # Old WordPress URLs -> new pages (Cloudflare Pages)
```

## Blog admin (`/admin`)

Publishing happens at **https://southseaswimrun.com/admin** — no local tools
needed. Sign in with a GitHub fine-grained personal access token that has
**Contents: read & write** on this repo (created under GitHub → Settings →
Developer settings → Fine-grained tokens). The token is stored only in that
browser.

What it does on publish (all in a single commit to `main`):

1. Generates the post page (`posts/<slug>.html`) from the text + photo gallery.
2. Uploads the photos (resized in the browser to max 1600 px JPEG).
3. Updates `posts/posts.json`, regenerates `blog.html`, the "latest posts"
   cards on `index.html` (between the `HOME-LATEST` markers) and `sitemap.xml`.

Cloudflare Pages then redeploys automatically (~1 minute).

Posts created before the admin are marked **legacy**: their listing details
can be edited and they can be deleted, but their article HTML stays as-is.

## Notes

- **Security headers** (CSP, HSTS, X-Frame-Options, etc.) live in
  `southsea-site/_headers`. The Content-Security-Policy is strict: no inline
  JS is allowed, so any new script must go in an external `.js` file.
  `/admin.html` has its own CSP that additionally allows `api.github.com`.
- **Copyright year** updates itself — use `<span data-year>2026</span>` and
  `js/main.js` rewrites it to the current year.
- **blog.html, sitemap.xml and the HOME-LATEST block in index.html are
  generated** from `posts/posts.json` by `js/templates.js`. Edit the manifest
  (or use /admin) instead of hand-editing those sections.
- **Team avatars:** drop square-ish photos at `images/team/vini.jpg`,
  `sarah.jpg`, `adrian.jpg`, `chris.jpg` (any size — they render at 72 px
  circles). Placeholders are in place until real photos land.
- **Images:** drop files into `images/` and reference them as `/images/<name>`.
